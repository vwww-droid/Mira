# TODO

## 本轮范围

本轮只做 Web Terminal(网页终端) 到真实 PTY(伪终端) 的连接能力, 作为 MVP(最小可行产品) 的唯一交付目标。

## 已记录但暂不实现的后续能力

1. Live UI(实时界面) 上传: 记录设备画面或交互结果上传需求, 本轮不做采集, 上传或回放。
2. 设备指标: 记录 CPU(中央处理器), 内存, 电量, 存储和网络状态等展示需求, 本轮不做采集。
3. logcat(Android 系统日志): 记录日志接入和过滤需求, 本轮不做解析或流式展示。
4. 自动化任务: 记录任务调度, 命令编排和历史执行需求, 本轮不做任务系统。
5. Probe(检测探针) 报告: 记录结构化风险报告需求, 本轮不做报告生成管线。
6. Agent(智能体) 分析: 已暴露 MCP(Model Context Protocol, 模型上下文协议) 工具接口, 本轮不实现自动决策, 评分或复查建议。
7. 多设备管理: 记录多设备发现, 切换和会话管理需求, 本轮只保留单浏览器会话到单 PTY 的闭环。

## 远程终端后续能力

1. apt(包管理器) 和小型包仓库: 记录包下发和索引管理需求, 本轮不维护软件源。
2. BusyBox(单文件工具集) 扩展: 已完成四 ABI(应用二进制接口) 会话临时释放和 manifest(清单), 后续补命令白名单, 版本切换和可观测日志。
3. AI Agent(智能体) 自动执行: 已新增 MCP(Model Context Protocol, 模型上下文协议) 工具接口, 后续补策略约束, 权限确认和批量任务编排。
4. 公网 Push(推送唤醒): 记录公网部署下的设备唤醒需求, 本轮只做设备主动反连和按需打开 PTY。
5. TLS(传输层安全协议) 和企业认证: 记录账号, 证书, HMAC(基于哈希的消息认证码) 等企业化认证需求, 本轮不实现共享令牌。
6. 多租户账号: 记录团队和租户隔离需求, 本轮默认自托管单服务端边界。

## 产品打磨 TODO

1. 手机首页 UI(用户界面) 产品化: 保持极简形态, 只输入 Relay URL(中继地址), 标题使用大号 `Mira` 和小号 `by vw2x`, 不展示解释性提示, 不暴露 Local Terminal(本地终端) 入口。
2. Relay(中继) 页面状态打磨: 设备在线, idle(空闲), opening(打开中), active(活跃), offline(离线) 状态要更直观。
3. 自动重连体验: Android control WebSocket(控制长连接) 已具备重连基础, 后续补更稳定的退避策略和无打扰状态同步。
4. 开源 README(项目说明) 包装: 做成产品首页风格, 突出一键公网启动, 手机填 URL(统一资源定位符), 浏览器打开终端的最小路径。

## 边界说明

1. 当前优先级只放在稳定打通浏览器和真实 PTY 的实时连接。
2. 后续能力仅作为方向记录, 不代表这些能力已经进入实现阶段。
3. 代码中不预留复杂框架, 避免第一版 MVP 被未来功能拖重。
